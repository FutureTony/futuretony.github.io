Interaction System

The Interaction system was a central part of the game as it was the main interaction between the player and the world. The system is composed of the script UComponentActor.cpp.
The UComponentActor script works by on button press it will fire a raycast from the player camera and check if the object hit is interactable, if it is it will call the apropreate event on the target object.
The reason why I opted for this system was that it gave the designers the freedom to customise the events that would trigger without needing to know how to code, as all the interactions where done in blueprints.

Save System

The Save system although not as central as the Interaction system, it still had its purpose.
The Save system was supposed to be a simple save system that would just keep track of what level the player had reached, but after some thought the team decided it needed to save the gamestate as well.
The Save system is made solely in blueprints. It uses a separete game instance, so that when the game got saved I would override all the data in the previous game instance with the current game data.
